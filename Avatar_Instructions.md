# 📂 分身并行开发指令集 (Avatar Mission Briefs - V2.0)

## ⚠️ 架构师全局警告 (Zero-Conflict & High Impact)
为了在最终合并时实现极具冲击力的演示，所有分身必须遵守以下**功能性差异化原则**。
禁止简单的“换皮”，必须在**物理属性、AI 逻辑或事件交互**上做出显著区别。

---

## 🏁 关卡过场与跳转契约 (Transition Protocol)
每一关必须包含一个**“终点旗杆” (Flagpole)**。
当马里奥触碰到旗杆时：
1. **触发通关动作**：旗杆降旗，马里奥滑下。
2. **拉起确认弹窗**：调用引擎内置的 `window.showTransition(nextLevelName)`。
3. **玩家确认**：玩家点击按钮后，才会加载并跳转至下一关。

## 🔵 关卡 A：深海迷踪 (Avatar: `water`)
*   **需求指令**：开发一个深海关卡，重写物理参数实现水下浮力。
*   **功能特性**：马里奥必须在水下通过“点按跳跃”实现类似 Flappy Bird 的推进，松开则缓慢下沉。
*   **视觉签名**：深蓝色调，珊瑚背景，终点旗杆被海藻缠绕。
*   **顺位**：`sequence: 1`

---

## 🔴 关卡 B：熔岩堡垒 (Avatar: `lava`)
*   **需求指令**：开发一个岩浆堡垒关卡，增加高侵略性 AI 怪物。
*   **功能特性**：怪物必须具备“预判性喷火”，且马里奥踩踏时移动更加沉重。增加“岩浆喷发”随机事件。
*   **视觉签名**：暗红色调，熔岩流动效果。
*   **顺位**：`sequence: 2`

---

## 🟡 关卡 C：云端之巅 (Avatar: `sky`)
*   **需求指令**：开发一个天空关卡，处理极端环境事件。
*   **功能特性**：加入“全局强力侧风”，马里奥必须时刻保持向右推杆才能抵消风力。平台应具有“响应式下坠”效果。
*   **视觉签名**：浅蓝/金色高空，云朵平台。
*   **顺位**：`sequence: 3`

## 📝 核心 API 调用指南
分身在开发关卡 JS 时，应利用以下全局钩子：

```javascript
// 在 checkWin 中检测与旗杆的碰撞
checkWin: function(player) {
    if (player.x > FLAGPOLE_X) {
        // 1. 触发滑竿动画逻辑 (由你实现)
        // 2. 拉起跳转弹窗 (由引擎提供)
        window.showTransition('lava'); // 传递下一关的标识符
        return true; 
    }
    return false;
}
```
